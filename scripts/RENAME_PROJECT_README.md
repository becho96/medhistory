# Инструкция по переименованию директории проекта

## Проблема

Кириллические символы в пути проекта (`История болезни`) вызывают проблемы с выполнением команд через встроенный терминал Cursor.

## Решение

Переименовать директорию проекта на латиницу: `medhistory-local`

## Безопасное переименование

### Способ 1: Использование скрипта (Рекомендуется)

1. **Откройте внешний терминал** (не встроенный Cursor):
   - Terminal.app, iTerm2 или другой

2. **Выполните скрипт**:
   ```bash
   cd "/Users/boris/Desktop/Начало/История болезни"
   bash scripts/rename-project-directory.sh
   ```

3. **Следуйте инструкциям** на экране

4. **После переименования**:
   - Закройте текущую папку в Cursor
   - Откройте новую: `File → Open Folder → /Users/boris/Desktop/Начало/medhistory-local`

### Способ 2: Вручную

1. **Проверьте статус Git** (через внешний терминал):
   ```bash
   cd "/Users/boris/Desktop/Начало/История болезни"
   git status
   ```

2. **Если есть незакоммиченные изменения**, закоммитьте их:
   ```bash
   git add .
   git commit -m "Save changes before rename"
   ```

3. **Переименуйте директорию**:
   ```bash
   cd "/Users/boris/Desktop/Начало"
   mv "История болезни" medhistory-local
   ```

4. **Откройте новую директорию в Cursor**:
   ```
   /Users/boris/Desktop/Начало/medhistory-local
   ```

5. **Проверьте, что все работает**:
   ```bash
   cd /Users/boris/Desktop/Начало/medhistory-local
   git status
   ls -la
   ```

## Что будет работать после переименования

✅ Все команды в встроенном терминале Cursor
✅ Git операции
✅ Docker команды
✅ npm/pip установки
✅ Любые скрипты из папки `scripts/`

## Что НЕ изменится

- Git история остается неизменной
- Remote репозиторий остается тем же (github.com:becho96/medhistory.git)
- Все ветки и коммиты сохраняются
- Содержимое файлов не меняется

## Проверка после переименования

В новой директории выполните:

```bash
# Проверка Git
git status
git log --oneline -5

# Проверка структуры проекта
ls -la

# Тест команды (должна работать без зависания)
pwd
```

## Возможные проблемы

### Если Cursor не видит новую папку
1. Полностью закройте Cursor (Cmd+Q)
2. Откройте заново
3. File → Open Folder → выберите новую папку

### Если Git показывает ошибки
Git не зависит от имени папки, все должно работать. Если есть проблемы:
```bash
git remote -v  # Проверьте remote
git branch     # Проверьте ветки
```
